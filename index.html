Website Creation & Deployment

A specialized guide for website creation and deployment. Follow the steps to move from an AI prompt to a professional live website.

1

Phase I: The Creative Spark (AI Interface)

The journey begins by initiating a new conversation within the Gemini interface. To unlock the full potential of code visualization, you must click on the Tools icon and specifically select Canvas. This environment is designed for collaborative coding and real-time previews, which is essential for ensuring your website looks exactly how you intend.

Before entering your instructions, ensure you switch the AI model from fast to thinking mode. This allows the model to process complex architectural requests with higher precision. You can then craft your prompt, such as asking Gemini to visualize a website based on an attached PDF or a set of rules. Upload any necessary files at this stage to provide the context needed for a professional result.

Once the initial draft is generated, the process moves into fine-tuning. You should interact with the AI to adjust visuals, color schemes, and layout structures until the end result meets your specific business standards.

2

Phase II: Local Management (macOS)

When you are satisfied with the digital preview, click on the Code tab within Gemini and copy the entire block of HTML. On your MacBook, open the TextEdit application to begin the local saving process. Create a new document and paste your code directly into the editor.

To save the file correctly, hold the COMMAND + S keys. It is vital to name this document exactly index.html so that web servers recognize it as your homepage.

After saving, navigate through your folders to your Favourites and select the TextEdit directory if prompted, or simply organize your work by creating a new folder with a fitting title. Placing your index.html file inside this specific folder keeps your project organized and ready for the cloud.

3

Phase III: Global Deployment (GitHub)

With your local file ready, log into your GitHub account and click the button to create a New Repository. Choose a name for your project; remember that this name will form the end of your public URL. It is good practice to add a brief description of the project and check the box to include a README file.

Once the repository is created, select the Add file button followed by Upload files. You can now choose the index.html file from the folder you created on your MacBook. Finalize this upload by clicking the green Commit changes button to save the file to the cloud.

The final configuration happens in the Settings tab of your repository. Scroll down to the Code and automation section and select Pages. Ensure the source is set to Deploy from a branch and that the branch is assigned to Main with the folder set to / (root).

After clicking Save, your website will be live. You can return to your repository overview to find the link and share your interactive creation with the world.
